# UI Direction

Wolfie should feel like a premium dark market-intelligence cockpit: dense, controlled, readable, branded, and operational.

## Theme

The entire app should match the dashboard theme:

- dark terminal-grade panels
- cyan/green/red status accents
- compact data density
- clear hover and selected states
- no marketing landing page
- no decorative-only hero
- no broken or dead visible controls

## Signal Console

Signal Console replaces the prior 3D Bot Thought direction.

Required layout:

- left: bot roster, avatars, presets, trading mode, allocation context
- center: decision flow, market pulse, signal stack, conflict board
- right: trade economics, risk gate, decision state, what changes the bot's mind
- lower panels: learning memory, self-healing, plain-English education/source freshness

Selecting different bots must change reasoning and signal emphasis.

## Bots

Bots must use distinct reusable avatars or robot-face assets. Avoid initials, generic squares, broken images, and clipped logos.

Bot visuals should communicate risk, status, allocation, and decision readiness without oversized or decorative-only art.

## Dashboard

Dashboard is the operating view:

- metrics update from starting capital, allocations, fees, and stored activity
- decision ledger opens a decision frame
- source panels use acquisition-ladder freshness labels
- performance chart controls update visible state

There should be no person-level disclosure card on the dashboard unless a user opens a profile.

## Plain-English Explanations

Technical terms exposed in UI need concise explanations. Examples:

- spread
- slippage
- volume confirmation
- liquidity
- volatility
- risk/reward
- net expectancy
- fee drag
- cooldown
- politician disclosure delay
- insider activity
- options contract
- margin interest

## Visual QA Rules

Before reporting completion:

- build the app
- run visual tests
- inspect desktop and mobile screenshots
- verify Signal Console renders
- verify old 3D Bot Thought is not the primary interface
- verify no forbidden visible strings appear
- keep localhost running and provide the review URL
