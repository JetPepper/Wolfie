# Owner Recovery And Master Dev

Owner recovery exists so the founder/operator is not locked out of development and recovery builds.

Master identity:

- Name: Wolfie Master Dev Operator
- Email: dev.operator@wolfie.local
- Role: MASTER_DEV_OWNER

Development and test builds allow a deterministic signed assertion generated by `createDevOwnerRecoveryAssertion()`. This is only accepted when the runtime environment is `development` or `test`.

Production customer builds do not unlock from a password alone and do not unlock from a simple environment flag. Production owner recovery requires `WOLFIE_ENABLE_OWNER_RECOVERY=true`, verification material, and a signed recovery assertion. This pass includes a production-safe signed assertion stub that fails closed unless the assertion and public verification material match the expected shape. No private signing key ships in the repo.

`WOLFIE_MASTER_DEV_PASSWORD` may be used for local QA identity checks, but password alone must never unlock production full runtime.

Owner recovery must never send trade data to Wolfie Cloud. Tests pin development/test recovery, production blocking, password-only blocking, signed assertion path, and zero-trade-data behavior.
